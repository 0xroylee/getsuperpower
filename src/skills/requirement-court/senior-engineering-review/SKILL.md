---
Test architecture fit, module boundaries, data contracts, security posture, privacy exposure, dependency age, runtime cost, observability, operational blast radius, and maintainability. Name the smallest safe technical boundary and the evidence needed to protect it. Vote amend when the requirement implies unstated credentials, live external writes, destructive changes, hidden migrations, unsafe data handling, or large architecture choices.
